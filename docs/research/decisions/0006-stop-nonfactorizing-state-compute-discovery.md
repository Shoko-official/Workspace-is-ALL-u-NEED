## Context

Decision 0005 left the project at `TERMINAL_STOP` because none of sixteen
revision, concurrency, migration, causal, longitudinal, or evaluation objects
survived. The repository owner nevertheless preserved a stricter long-term
objective: discover a genuinely original object that can direct future LLMs
toward governed persistent state and adaptive computation without rewriting or
assembling existing work.

M1d therefore excluded every M1c family, another joint budget router,
validity-before-compute restatements, benchmarks, taxonomies, manifests, and
novelty inferred from search silence. Three independent lanes generated 24
formal, learning-dynamics, and runtime candidates.

## Findings

### The behavioral comparison class was underidentified

At a fixed history `h`, the requested behavioral non-factorization property
cannot be established against an unrestricted state manager plus compute
router. For any joint distribution over memory action `m` and compute action
`c` on discrete or standard Borel spaces, the chain rule gives

\[
\pi(m,c\mid h)
=
\pi_M(m\mid h)\pi_C(c\mid h,m).
\]

The manager can sample `m` and transmit `(h,m)` to the router. The resulting
sequential composite reproduces the fixed-history law of `(m,c)` exactly. A
regular conditional is determined only `pi_M`-almost surely, and its values on a
`pi_M`-null set are immaterial; in the discrete case, zero-marginal actions may
be assigned arbitrary conditionals. This assumes unrestricted information
passage. It does not establish causal-order,
computational-cost, latency, private-information, learning, optimization, or
sample-efficiency equivalence.

A valid behavioral separation theorem would first have to restrict message
size, observations, decision order, feedback, training, shared randomness, or
computational class for an intrinsic reason. A causal, computational, or
learning claim instead needs its own matched comparator and estimand or bound.
None of the candidates supplied such a result.

This does not say that architecture and interfaces never matter. It says that a
joint-versus-separated behavioral comparison is not a technical novelty claim
until the separated class is formally specified and the restriction is part of
the scientific object.

### Agent-level positive records and composites

ReasoningBank/MaTTS studies a bidirectional interaction between memory and
test-time scaling: additional trajectories create contrastive signals for later
memory, and memory quality changes scaling returns. With Zuo et al., it supplies
the strongest positive composite for the proposed build-compute by read-compute
kernel, but neither record evaluates the exact held-out factorial. The
state-quality elasticity candidate is independently inadmissible because it
recycles the forbidden validity-before-compute route.

BudgetMem exposes low, medium, and high compute tiers for runtime memory modules
and trains one router from the query and intermediate memory state. AdaMEM
refreshes strategy memory during an episode across inference-compute levels,
without establishing the candidate threshold law. MemoPilot treats memory
updates as a multi-turn decision problem with delayed credit, without identifying
the proposed forced-reuse causal effect. TrustMem verifies candidate memory
transitions, while Memory-R2 shows that divergent writes create different future
environments and uses local rerollouts from a shared state. Together these are
positive composites and controls, not exact direct anticipation of every stated
technical object.

For durable-compute amortization, the existing Agent Memory systems
characterization supplies lifecycle construction, retrieval, generation,
freshness, and amortization measurements, while dynamic value of computation
supplies the lifetime objective. Their composite leaves no transportable law in
the candidate beyond ordinary amortized caching under a future-value model.

Useful Memories reports that repeated LLM consolidation can make memory utility
rise and then fall below a no-memory baseline. TriggerBench reports the effect
of reasoning load on prospective-memory recall and false alarms. Its one-
constraint Appendix C.3 regime reports no upstream AIME-accuracy perturbation;
it does not vary intention load or identify a positive reverse impairment. That
reverse candidate remains unsupported and non-distinct because it supplies no
mechanism, estimator, or bound beyond a generic shared-capacity account.

### Model and runtime collisions

CRAM distills episodic retrievals into semantic memory, lets consolidation
quality influence routing, and directly anticipates consolidation-driven
retirement of compute. Its authors also report decreasing attention demand and
state a lower bound against static routing on recurring patterns. The theorem
claim is not decision-critical because the preprint retains an unresolved macro
and an anonymized artifact.

INFERCEPT chooses among preserving, swapping, discarding, and recomputing
interrupted KV/context state. Preble jointly manages prefix state and compute
because both must be colocated for efficient inference. These systems partially
anticipate runtime continuation and jointly reconstruct residence scheduling;
they do not implement semantic read/write sets, randomness, receipts, or version
validity. The richer continuation nevertheless recycles the forbidden M1c replay
and repair family.

Proof-carrying code plus incrementally verifiable computation reconstruct generic
certificate-carrying state. SAVeR verifies and repairs beliefs before action
commitment, not an incremental dependency-cone certificate. The candidate is
excluded primarily because it recycles M1c certificate, dependency, and
revalidation objects. Separately, combining adaptive agreement with a memory
transition verifier reconstructs the dual-consensus barrier without adding a
new guarantee.

### Generic reductions

The remaining formal candidates reduce without LLM-specific residue:

- an anytime safe-write gate is an e-process or confidence sequence followed by
  a shield;
- lifetime persistent value of computation is represented by dynamic VoC on a
  correctly specified augmented meta-MDP state, without implying tractable
  identification or solution;
- compute-induced memory selection bias admits adaptive doubly robust correction
  only under recorded propensities, positivity, consistency, and sequential
  exchangeability; hidden-state OPE needs additional assumptions;
- compute-to-write mediation is a longitudinal stochastic path effect;
- deferred write credit under forced write and reuse is a longitudinal total
  effect or policy-value contrast under consistency, positivity, and sequential
  exchangeability;
- adaptive acquisition is adaptive submodular optimization when the required
  diminishing-return and feasibility hypotheses hold;
- persistent-state recourse is online optimization with predictions and
  switching costs under the stated convexity and regularity assumptions;
- obligation reservation is real-time scheduling; and
- epistemic compute is belief-state POMDP value of information.

The revocation frontier, state-quality factorial, proof-carrying state,
contamination-overcompute, self-sealing trap, and semantic continuation recycle
families explicitly excluded by the M1d issue. Validation debt and the reverse
prospective-intention effect provide unsupported notation or intuition without
a distinct theorem, estimator, mechanism, or bound.

### Search limitation

The search was targeted and current to 15 July 2026. It is not exhaustive and
does not claim that every relevant paper was found. The decision rests on
positive records, explicit strongest-composite reconstructions, conditional
generic reductions, and fixed-history behavioral factorization. Search silence
is not used.

## Decision

Return global `STOP` for all 24 M1d candidates.

- Zero candidates are `DISTINCT_CANDIDATE`.
- One is directly anticipated, six are anticipated by positive composites, nine
  reduce to known theory under their stated hypotheses, two are unsupported and
  not distinct, and six violate the preregistered no-recycling boundary.
- No model, benchmark, experiment, implementation, training run, paid compute,
  or manuscript is authorized.
- The M1d register and this decision are negative governance results, not the
  requested article.

The owner's direction-setting objective remains active at the program level.
This cycle does not satisfy it.

## Anti-rewrite determination

The following are explicitly insufficient for a future article:

- renaming dynamic value of computation as persistent or lifetime VoC;
- reporting a nonzero memory-quality by compute interaction after ReasoningBank;
- combining BudgetMem, AdaMEM, TrustMem, and MemoPilot into one architecture;
- presenting a POMDP with write, retrieve, verify, and halt actions as a new
  state-compute theory;
- treating confidence-sequence stopping, doubly robust estimation, causal
  mediation, adaptive submodularity, or dynamic regret as LLM-specific because
  state and compute labels were substituted;
- attaching proofs or provenance to memory without a new soundness or complexity
  result beyond proof-carrying and incremental systems;
- persisting a KV continuation and calling checkpoint/resume a semantic memory
  mechanism; or
- defining a weak independently trained router and treating its loss against a
  joint policy as evidence of intrinsic non-factorization.

## Alternatives considered

- `GO` was rejected because no candidate survived both positive prior art and
  generic reduction.
- `PIVOT` to lifetime VoC was rejected because dynamic VoC on a correctly
  specified augmented meta-MDP represents the proposed persistent externality;
  the candidate adds no identification, complexity, or solution result.
- `PIVOT` to compute-induced selection bias was rejected because adaptive DR
  supplies a correction under recorded propensities, positivity, consistency,
  and sequential exchangeability, while Memory-R2 supplies a fair shared-state
  comparison control. The candidate states no new identified hidden-state
  estimand outside those assumptions.
- `PIVOT` to a behavioral joint-versus-separated lower bound was rejected because
  no intrinsic restricted information, communication, computation, or learning
  class was proposed. The chain rule alone does not reject a future causal,
  complexity, or learning separation formulated inside such a class.
- Running one of the minimum experiments was rejected because a measured effect
  cannot create a new object after direct or composite anticipation, conditional
  generic reduction, or preregistered scope exclusion.
- A position, survey, or roadmap article was rejected by the owner's
  anti-rewrite requirement.
